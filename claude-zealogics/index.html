Part 2

設定員工的工作環境

Page 14

使用 Warp 終端機，加上 Dashboard

Warp 可以直接看到資料夾和 .md 文件，對我來說比一般終端機方便。

下方有請 Claude Code 自己新增 Dashboard（Token 用量和 Reset 時間），方便我了解他還可以工作多久。

Page 15

設定 Git 做版本紀錄

只需要用自然語言跟 Claude 溝通就好，例如：

  • 幫我推上去 GitHub，建立 Pages
  • 我要看之前的版本紀錄
  • 幫我把＿＿版本抓下來

Page 16

撰寫 Claude.md
確保遵循工作守則

每次打開 Claude Code 開始新對話，都會自動先讀 Claude.md，可以請 Claude Code 自己撰寫並持續更新。

內容要精簡、撰寫最核心的守則就好，不然很消耗 Token：

  • 你是誰、身份是什麼
  • Always / Never 清單
  • 目前執行的專案概覽

Page 17

用 /plan 事先做好計劃

Claude Code 有內建 Plan mode，可以輸入 /plan 或 Shift+Tab 切換。

好處是在執行比較複雜的任務前，能先看到 Claude 的思路跟規劃，不用等到執行完才發現跟期待有太大偏差。

Page 18

用 /brainstorming 釐清需求

如果是需求比較模糊的專案（例如：想做一套簡報模板），可以安裝 /brainstorming skills 一步步確認規格，撰寫產品文件。

Star 170k

Page 19

用 /caveman 節省 75% token 用量

安裝 /caveman skills，讓 Claude 只講重點別廢話，實測覺得英文的節省效果比較好，所以我現在都是跟 Claude 講英文。

Star 48.3k

Page 20

讓 Claude 寫工作日誌

自己建立 /daily-log skills，在工作告一段落後，讓 Claude 紀錄完成事項、決策項目、待追蹤項目。

Page 22

就像公司來了一位有很大潛力的新人，
你要先讓他知道你期待的品質，並設定目標和限制，
他才能在這些前提下交付讓你滿意的產出。

Page 23

撰寫 brand-guideline.md

有完整的設計系統，才能創造品牌信任感、兼顧設計與開發效率：

  • 品牌主色、功能色
  • 中英字體使用
  • 視覺風格與元素

Page 24

色彩系統

可以提供主色後，自動產出完整色盤、搭配的功能色。

Page 25

串接 Figma MCP

Figma MCP (Model Context Protocol) 可以讓 AI Agent 直接讀取、理解並操作 Figma 設計檔案中的結構、元件、圖層與變數。

例如：我能讓 Claude Code 把它生成的色彩系統，在 Figma 建立完整的色彩變數。

Page 26

字型系統

在 Google Font 上挑選自己喜歡的字型，或是請 Claude 推薦也可以。

Page 27

Icon

挑選開源 Icon 資料庫，自動根據內容選擇合適 Icon。

Page 28

撰寫 coding-guideline.md

用我粗淺的知識寫下的規範：

  • 新建專案時，HTML/CSS/JS 要分離
  • CSS 要自動抓 color token.css
  • CSS class 要語意化命名
  • CSS 單位為 px/rem，數值遵照 4px grid

跟員工一起創造了什麼

Page 30

讓溝通更有效的小技巧

視情況切換 model

用 /model 可以切換不同模型。簡單輕量的任務交給 Sonnet，複雜有規模的任務交給 Opus，才能事半功倍。

萬言比不上一圖

遇到比較難描述的情境或期待結果，直接貼圖片示意就好。

靈活運用相關指令

context >50% 用 /clear，執行中想問其他問題用 /btw。

Page 31

職涯諮詢官網

讓有興趣的人更了解我的服務價值、特色、流程。

Page 32

Instagram 貼文模板

根據我貼上的文章內容，自動套用合適文字樣式（螢光筆、引言、卡片）。

Page 33

互動簡報模板

建立常見的簡報版型，我只需要提供文字和圖片就能快速建立簡報。

Page 34

日記分析報告

Page 35

建立屬於自己的 Skills

確保 AI 每次執行任務都遵循相同的 SOP，提升一致性、降低幻覺，也不用重複下同樣的 prompt。

我根據 IG 貼文模板、簡報模板、日記分析報告，和對應的寫作規範，建立了 /ig-post、/slide、/journal-report Skills。

會害怕員工取代自己嗎

Page 37

目前不會。如果沒有我（老闆）的引導，員工自己根本做不到這些事，其中有非常多的品味判斷跟專業思考，都是由我來介入。

Page 38

從執行到決策者

容易被取代的設計師 能發揮價值的設計師

執行既定的設計流程 能根據現況和資源做應變
只理解表層工具應用 認知到工具只是手段之一
溝通只停留在設計視角 同時考量商業、工程、使用者
抗拒跟害怕使用 AI 把 AI 當作放大影響力的協作夥伴
